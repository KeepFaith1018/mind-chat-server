## 2. 后端已实现功能清单

## 2.1 认证与登录

- 邮箱注册/登录/退出
- 刷新 Token（Cookie 模式）
- 获取当前登录用户
- OAuth 登录（GitHub / Google）并重定向前端

## 2.2 会话与消息

- 会话创建、列表、详情、更新、删除
- 消息历史按时间顺序读取

## 2.3 AI 聊天与能力开关

- SSE 流式回答（thinking/content/done/error）
- 支持能力开关：
  - `webSearch`
  - `reasoning`
  - `fileQa`
- 模型能力校验（不支持能力时拒绝或降级）

## 2.4 模型配置与动态读取

- 模型配置来自 `model_configs`
- 前端初始化可拉取可用模型与能力矩阵
- 默认模型自动回落

## 2.5 文件上传问答

- 支持上传文件并在对话中引用
- 10MB 大小限制

## 2.6 联网搜索（已接入）

- Chat 链路可触发检索并注入上下文
- SSE 通过 `tool_result` 回传检索来源

## 2.7 计费与配额

- 每轮对话写入 `usage_logs`
- 成本按模型单价计算
- 配额按 token 统计并拦截超限

## 2.8 安全与限流

- 输入安全检测（Prompt Injection 关键词）
- 输出安全检测（危险片段拦截）
- 接口限流（进程内，用户/IP 维度）

## 2.9 对话分享（新增）

- 生成分享链接
- 公开读取分享 JSON
- 公开静态只读页面渲染消息

## 3. 接口总览（BaseURL: `/v1`）

说明：除 SSE/重定向/HTML 外，其它接口走统一响应包装。

## 3.1 Auth

- `POST /auth/register`
- `POST /auth/login`
- `POST /auth/logout`
- `POST /auth/refresh`
- `GET /auth/me`（需登录）
- `GET /auth/oauth/:provider`
- `GET /auth/:provider/callback`

## 3.2 Models

- `GET /models/available`

## 3.3 Conversations

- `POST /conversations`（需登录）
- `GET /conversations`（需登录）
- `GET /conversations/:id`（需登录）
- `PATCH /conversations/:id`（需登录）
- `DELETE /conversations/:id`（需登录）

## 3.4 Chat

- `POST /chat`（需登录，SSE）
- `GET /chat/capabilities`（需登录）

## 3.5 Upload

- `POST /upload`（需登录，multipart/form-data）

## 3.6 Share

- `POST /share`（需登录）  
  创建分享链接，body: `{ conversationId, expireHours? }`
- `GET /share/:linkId`（公开）  
  获取分享内容 JSON
- `GET /share/:linkId/page`（公开）  
  获取静态只读 HTML

## 4. 前端重点对接说明

## 4.1 登录态与请求基线

1. 前端请求统一带 `credentials: 'include'`
2. 启动时调用 `GET /auth/me` 恢复登录态
3. 401 时优先尝试 `POST /auth/refresh`，再重试原请求

## 4.2 聊天页核心流程

1. 初始化拉取 `GET /models/available`
2. 渲染模型下拉 + 能力开关（根据 capability 决定是否可点击）
3. 发起 `POST /chat`（SSE）并监听：
   - `meta`
   - `tool_result`
   - `thinking`
   - `content`
   - `done`
   - `error`
4. 对话结束后刷新会话列表和详情

## 4.3 分享功能前端流程

1. 在会话页点击“分享”
2. 调用 `POST /share` 生成链接
3. 展示 `shareUrl` 并提供复制按钮
4. 外部访问 `.../share/:linkId/page` 只读浏览

## 5. SSE 事件协议（前端必须支持）

## 5.1 `meta`

字段：

- `conversationId`
- `model`
- `capabilities`（实际生效能力）

## 5.2 `tool_result`

联网搜索结构（v1）：

- `version`
- `tool`
- `query`
- `total`
- `truncated`
- `generatedAt`
- `items[]`（rank/title/url/snippet）

## 5.3 `thinking` / `content`

- `thinking.delta`：推理增量
- `content.delta`：回复增量

## 5.4 `done` / `error`

- `done`：流完成
- `error.message`：错误提示

## 7. 前端优化建议（按优先级）

## P0（尽快）

1. SSE 断线重连与超时提示
2. 分享链接复制成功反馈
3. 能力开关禁用态（模型不支持时置灰 + tooltip）
4. 错误码到文案映射（如 46001 配额不足）

## P1（建议）

1. `tool_result` 来源卡片 UI（可点击原链接）
2. 聊天消息虚拟列表（长会话性能）
3. 会话列表乐观更新与回滚
4. 分享页品牌化样式（标题、头像、时间格式）

## P2（可选）

1. 分享链接过期时间选择器（1h/24h/7d）
2. 分享页导出 markdown
3. 前端自渲染分享页（替代后端静态 HTML）

## 9. 当前后端已知注意点

1. 限流为进程内实现，适合单实例环境
2. 分享页面为后端直出 HTML，适合快速上线
3. 若部署在公网，建议配置 `APP_URL` 生成正确分享地址
4. 新增分享表后需执行对应迁移 SQL
