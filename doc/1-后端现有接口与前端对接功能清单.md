# 后端现有接口与前端对接功能清单

## 1. 文档目标

本文档基于当前 NestJS 后端代码，梳理前端可直接对接的能力，包括：

- 认证与登录（邮箱/密码、GitHub/Google OAuth）
- 用户信息与登录态维护
- 会话管理（列表、详情、更新、删除）
- AI 对话（SSE 流式返回）
- 文件上传与文件内容注入问答

---

## 2. 基础约定

### 2.1 服务地址与前缀

- 本地后端默认端口：`http://localhost:3001`
- 全局前缀：`/v1`
- 前端对接 Base URL：`http://localhost:3001/v1`

### 2.2 CORS 与 Cookie

- 后端允许来源：`http://localhost:3000`
- 已开启 `credentials: true`
- 前端请求必须带 `credentials: 'include'`

### 2.3 统一响应结构（非重定向、非SSE）

成功：

```json
{
  "success": true,
  "code": 0,
  "message": "success",
  "data": {}
}
```

失败：

```json
{
  "success": false,
  "code": 40100,
  "message": "未登录"
}
```

### 2.4 鉴权规则

- 登录成功后后端写入 Cookie：
  - `token`（15分钟）
  - `refresh_token`（7天）
- 鉴权优先读取 Cookie，也支持 `Authorization: Bearer <token>`

---

## 3. 前端可实现的功能总览

1. 邮箱注册/登录/退出
2. OAuth 登录（GitHub、Google）
3. 登录态恢复（`/auth/me` + `/auth/refresh`）
4. 会话 CRUD（创建、列表、详情、编辑、删除）
5. SSE 流式对话（含 `thinking/content/done/error` 事件）
6. 上传 PDF/Word/TXT/Markdown 并在对话中引用文件内容

## 4. 接口明细

## 4.1 认证模块 `Auth`

### 4.1.1 邮箱注册

- 方法/路径：`POST /auth/register`
- 鉴权：否
- Body(JSON)：

```json
{
  "email": "user@example.com",
  "password": "123456",
  "name": "Faith"
}
```

- 前端能力：注册并自动登录（后端会设置 Cookie）

### 4.1.2 邮箱登录

- 方法/路径：`POST /auth/login`
- 鉴权：否
- Body(JSON)：

```json
{
  "email": "user@example.com",
  "password": "123456"
}
```

- 前端能力：登录并落地 Cookie 登录态

### 4.1.3 退出登录

- 方法/路径：`POST /auth/logout`
- 鉴权：否
- Body：无
- 前端能力：退出，清理登录态

### 4.1.4 刷新 Token

- 方法/路径：`POST /auth/refresh`
- 鉴权：否（依赖 `refresh_token` Cookie）
- Body：无
- 前端能力：Access Token 过期后自动续期

### 4.1.5 获取当前用户

- 方法/路径：`GET /auth/me`
- 鉴权：是
- 前端能力：页面初始化恢复用户信息

### 4.1.6 发起 OAuth 登录

- 方法/路径：`GET /auth/oauth/:provider?redirect=/chat`
- provider：`github` | `google`
- 鉴权：否
- 返回：302 重定向到第三方授权页
- 前端调用示例：

```ts
window.location.href =
  `http://localhost:3001/v1/auth/oauth/github?redirect=${encodeURIComponent('/chat')}`;
```

### 4.1.7 OAuth 回调（后端处理）

- 方法/路径：`GET /auth/:provider/callback`
- 鉴权：否
- 返回：302 跳转前端地址（默认 `FRONTEND_URL` 或 `http://localhost:3000`）
- 前端能力：OAuth 登录完成后自动回到业务页面

## 4.2 会话模块 `Conversations`

### 4.2.1 创建会话

- 方法/路径：`POST /conversations`
- 鉴权：是
- Body(JSON)：

```json
{
  "modelId": "deepseek-ai/DeepSeek-V3"
}
```

### 4.2.2 会话列表（分页）

- 方法/路径：`GET /conversations?page=1&pageSize=20`
- 鉴权：建议带登录态
- 返回字段：`items/total/page/pageSize/totalPages`

### 4.2.3 会话详情（含消息）

- 方法/路径：`GET /conversations/:id`
- 鉴权：建议带登录态

### 4.2.4 更新会话

- 方法/路径：`PATCH /conversations/:id`
- 鉴权：建议带登录态
- Body(JSON)：

```json
{
  "title": "新的会话标题",
  "isPinned": true,
  "modelId": "deepseek-ai/DeepSeek-V3"
}
```

### 4.2.5 删除会话

- 方法/路径：`DELETE /conversations/:id`
- 鉴权：建议带登录态
- 返回：`{ success: true }`

## 4.3 聊天模块 `Chat`（SSE）

### 4.3.1 发起聊天（流式）

- 方法/路径：`POST /chat`
- 鉴权：是
- Header：`Accept: text/event-stream`
- Body(JSON)：

```json
{
  "messages": [
    { "role": "user", "content": "你好" }
  ],
  "model": "deepseek-ai/DeepSeek-V3",
  "conversationId": "可选",
  "fileIds": ["可选文件id"]
}
```

- 返回：`text/event-stream`
- 事件协议：
  - `meta`：`{ conversationId }`
  - `thinking`：`{ delta }`
  - `content`：`{ delta }`
  - `done`：`[DONE]`
  - `error`：`{ message }`

前端消费示例（伪代码）：

```ts
const es = await fetch('/v1/chat', { method: 'POST', credentials: 'include', body: JSON.stringify(payload) });
// 按 event 字段分发处理 thinking/content/done/error
```

## 4.4 文件上传模块 `Upload`

### 4.4.1 上传文件

- 方法/路径：`POST /upload`
- 鉴权：是
- Content-Type：`multipart/form-data`
- 字段：`file`
- 最大体积：10MB
- 支持类型：PDF、Word(doc/docx)、TXT、Markdown
- 返回示例：

```json
{
  "id": "file_id",
  "filename": "需求说明.pdf",
  "size": 102400,
  "contentPreview": "文本前200字符..."
}
```

- 前端能力：上传后拿到 `id`，在 `/chat` 请求中传入 `fileIds`

---

## 5. 前端对接建议流程

1. 应用启动：调用 `GET /auth/me` 判断登录态
2. 未登录时：支持邮箱登录或 OAuth 登录
3. 登录后：先拉 `GET /conversations` 渲染侧栏
4. 进入会话：`GET /conversations/:id` 回显历史消息
5. 发送消息：`POST /chat`，按 SSE 增量渲染回复
6. 上传文件：`POST /upload` -> 得到 fileId -> 下次 chat 带上 `fileIds`
7. 401 场景：先调 `POST /auth/refresh`，成功后重试原请求

---

## 6. 已知事项（当前代码现状）

1. OAuth 成功后会重定向到前端域名，需配置 `FRONTEND_URL`（建议 `.env` 配置为 `http://localhost:3000`）。
2. 前端调用 OAuth 跳转建议使用完整 URL，避免相对路径解析问题。
3. 聊天接口是 SSE，不走标准 JSON 包装，前端需按事件流处理。

## 7. 常用错误码

- `40000` 参数错误
- `40100` 未登录
- `40101` 登录已过期
- `40400` 资源不存在
- `42001` 文件类型不支持
- `42002` 文件不存在
- `43001` 会话不存在
- `46001` 配额不足
- `47001` 账号或密码错误
- `47002` 用户已存在
- `47003` 用户不存在
- `47004` 刷新令牌无效
- `50000` 服务器内部错误
