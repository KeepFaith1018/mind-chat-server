# MindChat 认证系统实施方案 (Auth Implementation Plan)

> **版本**: 1.0.0
> **日期**: 2026-03-12
> **目标**: 实现一套符合 Soft UI 风格、安全且具备多平台登录能力的认证系统。

---

## 一、 视觉设计 (UI Design)

基于 [FRONTEND_DESIGN.md](file:///Users/faith/WorkSpace/MindChat/doc/FRONTEND_DESIGN.md) 的 Soft UI 理念：

### 1.1 页面布局

- **路由**: `/login` (同时支持登录与注册切换)。
- **布局**: `layout: false` (全屏居中，与 Landing Page 风格统一)。
- **背景**: 延续 `bg-white dark:bg-gray-950`，配合微弱的渐变光晕 (Glow Effect)。

### 1.2 核心组件样式

- **Auth Card**:
  - 宽度 `max-w-md`，圆角 `rounded-3xl`，投影 `shadow-2xl`。
  - 背景 `bg-white/80 dark:bg-gray-900/80` (配合 `backdrop-blur`)。
- **Tabs**:
  - 使用 Nuxt UI 的 `UTabs`，圆角 `rounded-2xl`，切换动画丝滑。
- **Inputs**:
  - 圆角 `rounded-xl`，背景 `bg-gray-50 dark:bg-gray-800`，无边框，仅在聚焦时显示 `ring-primary-500`。
- **Social Buttons**:
  - 按钮尺寸增大，圆角 `rounded-xl`，使用 Lucide 图标。

## 二、 技术实现流程 (Technical Flow)

### 2.1 路由与中间件拦截

1. **访问首页 (`/`)**: 纯展示 Landing Page。
2. **点击“立即开始对话”**: 跳转至 `/chat`。
3. **[auth.global.ts](file:///Users/faith/WorkSpace/MindChat/app/middleware/auth.global.ts) 拦截**:
   - 若 `!isLoggedIn` -> 重定向至 `/login?redirect=/chat`。
4. **登录成功后**: 跳转回 `redirect` 参数指定的页面 (默认为 `/chat`)。

### 2.2 前端状态管理 (Pinia)

更新 [useAuthStore.ts](file:///Users/faith/WorkSpace/MindChat/app/stores/useAuthStore.ts):

- 移除 Mock 登录逻辑。
- 实现 `login(email, password)`: 调用 `POST /api/auth/login`。
- 实现 `register(email, password, name)`: 调用 `POST /api/auth/register`。
- 实现 `fetchUser()`: 调用 `GET /api/auth/me` 并更新 `user` 状态。
- **持久化**: 使用 `useStorage` 保持 Token，并在 `onMounted` 或中间件中初始化用户信息。

### 2.3 后端 API 职责

- **[login.post.ts](file:///Users/faith/WorkSpace/MindChat/server/api/auth/login.post.ts)**: 验证密码 (bcryptjs)，返回 JWT。
- **[register.post.ts](file:///Users/faith/WorkSpace/MindChat/server/api/auth/register.post.ts)**: 创建用户，返回 JWT。
- **OAuth (GitHub/Google)**:
  - 路由: `/api/auth/github` & `/api/auth/google`。
  - Callback 处理: 查找/创建 `User` 和 `Account` (Prisma)，重定向回前端带上 Token。
