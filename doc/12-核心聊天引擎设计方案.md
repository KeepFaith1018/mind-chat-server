# 核心聊天引擎设计方案 (LangChain + 硅基流动)

## 1. 概述

本方案旨在构建 MindChat 的核心聊天引擎，移除原有的 OpenAI SDK，全面采用 **LangChain** 框架对接 **硅基流动 (SiliconFlow)** 的大模型服务。

### 核心目标

1.  **模型解耦**：通过 LangChain 统一接口，支持无缝切换 DeepSeek-V3, DeepSeek-R1, Qwen2.5 等多种模型。
2.  **流式响应**：实现 Server-Sent Events (SSE) 协议，支持打字机效果。
3.  **思考模式支持**：适配 DeepSeek R1 的深度思考 (Thinking) 内容展示。
4.  **上下文管理**：基于数据库的历史消息自动构建会话上下文。

---

## 2. 技术架构

### 2.1 后端技术栈

- **Runtime**: Nuxt 4 Server (Nitro)
- **AI Framework**: `@langchain/openai`, `@langchain/core`
  - _注：SiliconFlow 兼容 OpenAI 协议，因此使用 `@langchain/openai` 即可，只需配置 BaseURL。_
- **Database**: Prisma (PostgreSQL)
- **Stream Protocol**: Standard SSE (text/event-stream)

### 2.2 数据模型 (已就绪)

基于现有的 `Conversation` 和 `Message` 模型：

- `Conversation`: 存储会话元数据 (标题、模型ID、置顶状态)。
- `Message`: 存储消息内容、角色、思考过程 (`reasoning_content`)、Token 消耗。

---

## 3. 详细设计

### 3.1 后端 API 设计

#### 3.1.1 聊天补全接口

- **Endpoint**: `POST /api/chat`
- **Auth**: Required (Bearer Token / Cookie)
- **Request Body**:
  ```json
  {
    "conversationId": "uuid-optional", // 如果为空则视为新会话（暂存状态）
    "model": "deepseek-ai/DeepSeek-V3",
    "messages": [{ "role": "user", "content": "你好" }]
  }
  ```
- **Response**: `text/event-stream`
  - 事件流格式：
    - `event: content` -> `data: {"delta": "..."}` (普通内容)
    - `event: reasoning` -> `data: {"delta": "..."}` (思考内容)
    - `event: error` -> `data: {"code": "...", "message": "..."}`
    - `event: done` -> `data: "[DONE]"`

#### 3.1.2 会话管理接口

- `GET /api/conversations`: 获取会话列表 (分页)。
- `GET /api/conversations/:id`: 获取单会话详情及历史消息。
- `POST /api/conversations`: 创建新会话。
- `DELETE /api/conversations/:id`: 删除会话。
- `PATCH /api/conversations/:id`: 修改标题/置顶。

### 3.2 核心业务流程

1.  **用户发起请求**：前端发送当前对话上下文 + 模型参数。
2.  **后端处理 (`/api/chat`)**：
    1.  **鉴权**：校验用户登录状态。
    2.  **配额检查**：检查用户 Token 配额 (可选)。
    3.  **上下文构建**：
        - 如果是已有会话 (`conversationId` 存在)，从数据库加载最近 N 条历史消息。
        - 将消息转换为 LangChain 格式 (`HumanMessage`, `AIMessage`, `SystemMessage`)。
    4.  **模型调用**：
        - 根据 `model` 参数初始化 `ChatOpenAI` 实例 (指向 SiliconFlow)。
        - 调用 `.stream()` 方法获取流。
    5.  **流式响应与持久化**：
        - **实时响应**：将流数据通过 SSE 推送给前端。
        - **异步入库**：在流结束后，将完整的 User Message 和 AI Message (含 reasoning) 存入数据库。
        - **Token 统计**：更新用户的 Token 使用量。

### 3.3 前端设计 (Store & UI)

#### 3.3.1 Pinia Store (`useChatStore`)

- **State**:
  - `conversations`: 会话列表。
  - `activeConversationId`: 当前会话 ID。
  - `messages`: 当前会话的消息列表。
  - `streamingMessage`: 正在接收的流式消息 (临时状态)。
  - `isLoading`: 是否正在生成。
- **Actions**:
  - `sendMessage(content, model)`: 发送消息，处理乐观更新 (Optimistic UI)。
  - `abortGeneration()`: 中断生成。

#### 3.3.2 SSE 客户端

- 使用 `fetch` 原生 API 或 `@microsoft/fetch-event-source` (推荐后者，支持 POST 和自定义 Header)。
- 处理自定义事件 (`content`, `reasoning`)，分别更新消息的正文和思考部分。

## 4. 实施步骤

### 第一阶段：后端核心 (Backend Core)

1.  **工具封装**：创建 `server/utils/ai.ts`，封装 LangChain 模型初始化逻辑，支持 SiliconFlow 配置。
2.  **流式接口**：实现 `server/api/chat/index.post.ts`，打通 "接收请求 -> 调用模型 -> 返回流" 的链路。
3.  **调试验证**：使用 Curl 或 Postman 验证流式输出是否正常。

### 第二阶段：数据持久化 (Persistence)

1.  **数据库操作**：实现消息的异步保存逻辑 (Prisma)。
2.  **会话管理**：实现会话列表、新建、删除 API。
3.  **Token 计费**：实现 Token 消耗统计与用户配额更新。

### 第三阶段：前端对接 (Frontend Integration)

1.  **Store 升级**：重构 `useChatStore` 以支持流式更新。
2.  **聊天界面**：
    - 实现 Markdown 渲染 (支持代码高亮)。
    - 实现 "思考中" 折叠面板 (针对 DeepSeek R1)。
    - 自动滚动与停止生成功能。
3.  **模型切换**：在 UI 上添加模型选择器，动态传递 `model` 参数。

## 5. 关键代码示例 (规划)

### 5.1 LangChain 初始化 (server/utils/ai.ts)

```typescript
import { ChatOpenAI } from '@langchain/openai'

export const createChatModel = (modelName: string) => {
  const config = useRuntimeConfig()
  return new ChatOpenAI({
    modelName,
    openAIApiKey: config.siliconFlowApiKey,
    configuration: {
      baseURL: 'https://api.siliconflow.cn/v1'
    },
    streaming: true
  })
}
```

### 5.2 前端流式处理 (useChatStore.ts)

```typescript
// 伪代码
await fetchEventSource('/api/chat', {
  method: 'POST',
  body: JSON.stringify({ messages, model }),
  onmessage(msg) {
    if (msg.event === 'content') {
      currentMessage.content += JSON.parse(msg.data).delta
    } else if (msg.event === 'reasoning') {
      currentMessage.reasoning += JSON.parse(msg.data).delta
    }
  }
})
```
