# MindChat 认证系统与 UI 骨架设计方案 (Day 2 & Day 3)

**日期**: 2026-03-11 (Updated)
**作者**: AI Assistant

## 1. 概述

本文档详细描述 MindChat 项目 Day 2 (Auth) 和 Day 3 (UI Skeleton) 的技术实现方案。
基于 Nuxt 4 + Nuxt UI (Tailwind CSS) + Prisma + PostgreSQL 技术栈。
包含服务端 Auth API 的实现，集成 GitHub/Google OAuth 2.0 登录。

## 2. Day 2: 认证系统 (Authentication)

### 2.1 依赖安装

需要安装以下依赖以支持认证流程：

```bash
# 状态管理
pnpm add pinia @pinia/nuxt

# 加密与 JWT (使用 jose 替代 jsonwebtoken)
pnpm add bcryptjs jose
pnpm add -D @types/bcryptjs

# UI 库
pnpm add @nuxt/ui
```

### 2.2 服务端 API 设计 (Server API)

文件位置: `server/api/auth/`

#### 2.2.1 注册接口 (`POST /api/auth/register`)

- **Request**: `{ email, password, name? }`
- **Logic**:
  1. 校验邮箱是否已存在。
  2. 使用 `bcryptjs` 对密码进行 hash。
  3. Prisma 创建 `User` 记录。
  4. 生成 JWT Token (使用 `jose`)。
- **Response**: `{ user: User, token: string }`

#### 2.2.2 登录接口 (`POST /api/auth/login`)

- **Request**: `{ email, password }`
- **Logic**:
  1. 根据邮箱查找用户。
  2. 使用 `bcryptjs.compare` 校验密码。
  3. 生成 JWT Token (使用 `jose`)。
- **Response**: `{ user: User, token: string }`

#### 2.2.3 获取当前用户 (`GET /api/auth/me`)

- **Headers**: `Authorization: Bearer <token>`
- **Logic**:
  1. 解析 Header 中的 Token。
  2. 使用 `jose.jwtVerify` 校验 Token 有效性。
  3. 返回对应的用户信息。
- **Response**: `{ user: User }`

#### 2.2.4 GitHub OAuth (`GET /api/auth/github`)

- **Logic**:
  1. 生成 `state`。
  2. 构建 GitHub OAuth URL (`https://github.com/login/oauth/authorize`)。
  3. 重定向用户。

#### 2.2.5 GitHub Callback (`GET /api/auth/github/callback`)

- **Query**: `code`, `state`
- **Logic**:
  1. 使用 `code` 向 GitHub 换取 `access_token`。
  2. 使用 `access_token` 获取 GitHub 用户信息。
  3. 查找 `accounts` 表中是否存在该 GitHub 用户。
  4. 若存在 -> 获取关联 User -> 生成 JWT -> 重定向回首页 (带 Token)。
  5. 若不存在 -> 创建 User (若 Email 未占用) -> 创建 Account -> 生成 JWT -> 重定向。

### 2.3 前端状态管理 (Pinia Store)

文件位置: `stores/auth.ts`

- **State**:
  - `user`: 当前登录用户信息 (User | null)
  - `token`: JWT Token (string | null)
- **Actions**:
  - `setUser(user)`
  - `setToken(token)`
  - `logout()`: 清除状态和 Cookie/LocalStorage

### 2.4 Composable: `useAuth`

文件位置: `composables/useAuth.ts`

封装 `useAuthStore`，提供统一的认证操作接口：

- `login(credentials)`: 调用 API 并更新 Store
- `register(credentials)`: 调用 API 并更新 Store
- `loginWithGithub()`: 跳转到 `/api/auth/github`
- `logout()`: 处理登出逻辑（包括路由跳转）
- `initAuth()`: 应用启动时从 Cookie 恢复状态

### 2.5 路由守卫 (Middleware)

文件位置: `middleware/auth.global.ts`

- **逻辑**:
  1. 定义公开路由白名单 (如 `/login`, `/register`).
  2. 检查用户是否已登录 (Store 中是否有 token/user).
  3. 若未登录且访问非白名单页面 -> 重定向至 `/login`.
  4. 若已登录且访问 Auth 页面 -> 重定向至 `/` (或 `/chat`).

## 3. Day 3: UI 骨架 (UI Skeleton)

### 3.1 依赖安装

```bash
pnpm add @nuxt/ui
```

### 3.2 布局设计 (Layouts)

#### 3.2.1 认证布局 (`layouts/auth.vue`)

- **结构**: 全屏居中容器，用于放置登录/注册表单。
- **组件**: 使用 `UCard` 作为容器，`UInput`, `UButton` 构建表单。

#### 3.2.2 主布局 (`layouts/default.vue`)

- **结构**: 经典的 Sidebar + Main Content 布局。
- **实现**:
  - **Sidebar**: 固定宽度 (260px)，左侧停靠，使用 `UVerticalNavigation`。
  - **Main**: `flex-1`，右侧自适应区域。

### 3.3 关键组件 (Components)

#### 3.3.1 侧边栏 (`components/layout/Sidebar.vue`)

- **功能**:
  - Logo 展示
  - "新建对话" 按钮 (`UButton`)
  - 历史对话列表 (`UVerticalNavigation` 或自定义列表)
  - 用户个人信息/设置入口 (`UDropdown` + `UAvatar`)

#### 3.3.2 顶部栏 (`components/layout/Header.vue`)

- **功能**:
  - 模型选择器 (`USelectMenu`)
  - 移动端菜单 Toggle 按钮 (`UButton` icon="i-heroicons-bars-3")

#### 3.3.3 聊天输入框 (`components/chat/ChatInput.vue`)

- **功能**:
  - 自适应高度 Textarea (`UTextarea` resize)
  - 发送按钮 (`UButton` icon="i-heroicons-paper-airplane")
  - 附件/工具按钮占位
