> **日期**: 2026-03-12
> **作者**: 前端架构组
> **状态**: 待确认

## 一、 设计概述

本方案基于 1.2.0 版本迭代，核心目标是**消除白屏焦虑**并**柔化视觉风格**。我们将从“线框硬朗风格”转向“现代简约圆润风格” (Soft & Minimalist)，并通过全局加载动画提升首屏体验。

### 1.1 核心变更点

1.  **加载体验 (Loading Experience)**:
    - 增加 App 启动时的全局 Loading 动画 (Logo 呼吸/脉冲效果)，消除 SSR 水合前的白屏。
    - 页面切换/对话加载增加骨架屏 (Skeleton) 或微交互。
2.  **视觉柔化 (Soft UI)**:
    - **去线框化**: 大量减少 `border` 的使用，改用柔和的背景色区分 (`bg-gray-50/50`) 或微投影 (`shadow-sm`)。
    - **圆角升级**: 全局圆角从 `rounded-md/lg` 提升至 `rounded-2xl` 或 `rounded-3xl`，营造亲和力。
    - **留白增加**: 增加组件间距，提升呼吸感。
3.  **布局调整 (同 1.2.0)**:
    - 侧边栏重构 (用户菜单下沉)。
    - 主页双形态 (居中/底部输入框)。
    - 模型选择下沉。

## 二、 视觉风格指南 (Visual Style Guide)

### 2.1 形状与质感

- **卡片/容器**: 使用 `rounded-2xl` 或 `rounded-3xl`。
- **按钮/输入框**: 使用全圆角 `rounded-full` 或大圆角 `rounded-xl`。
- **分隔**: 避免使用 `<hr>` 或显眼边框。使用 `gap` (间距) 或极淡的背景色块区分内容。
- **阴影**: 仅在悬浮或模态框使用 `shadow-lg`，默认状态追求扁平或低层级投影。

### 2.2 配色策略

- **背景**:
  - Page: `bg-white` / `dark:bg-gray-950`
  - Sidebar: `bg-gray-50/50` / `dark:bg-gray-900/50` (半透明/模糊)
  - Active Item: `bg-primary-50` / `dark:bg-primary-900/10` (淡色背景代替边框高亮)

## 四、 关键组件设计详情

### 4.1 全局加载 (Global Loading)

- **实现方式**: 在 `app.vue` 或 `layouts/default.vue` 最外层添加覆盖层。
- **触发**: `nuxtApp.hook('page:start')` / `page:finish` 或首屏挂载前。
- **动画**: Logo 居中呼吸，或简单的 3-dots 跳动。
- **目的**: 掩盖资源加载和水合过程。

### 4.2 侧边栏 (Sidebar) - [已实现基础，需重构]

- **风格**: 移除右侧边框，使用背景色区分。
- **用户菜单**: 移至底部，点击弹出 `UDropdown` (圆角 `rounded-2xl`，阴影 `shadow-xl`)。

### 4.3 输入框 (Input Box) - [已实现基础，需重构]

- **形态**:
  - **Centered**: 大阴影 `shadow-xl`，背景纯白/深灰，圆角 `rounded-3xl`。
  - **Bottom**: 悬浮感或玻璃拟态，不再与底部边缘硬连接。
- **交互**: 聚焦时外发光环 (Ring) 颜色更柔和。
