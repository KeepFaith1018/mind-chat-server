# MindChat 需求分析与实现计划 (Vue 3 + Nuxt 4)

> **目标**：Vue 3 + Nuxt 4，打造一个具备高性能流式对话、复杂状态管理、并行工具调用的 AI 聊天平台。作为简历亮点项目，重点突出全栈开发能力、性能优化及复杂业务逻辑处理。

---

## 一、 需求分析 (Requirement Analysis)

### 1. 核心功能 (MVP - 必做)

为了保证项目能尽快上线并具备基本使用价值，以下功能为第一优先级：

- **用户认证系统 (Auth)**:
  - 基于 JWT 的邮箱/密码注册登录。
  - Nuxt Middleware 路由守卫（保护 `/chat` 路径）。
  - 简单的 OAuth2.0 (GitHub/Google) 集成。
- **SSE 流式聊天 (Core SSE)**:
  - 实时流式接收 AI 回复。
  - 支持 **Thinking** 过程展示（类似 DeepSeek/OpenAI 推理模型）。
  - 支持 **Tool Calling** 进度反馈。
- **消息生命周期管理 (State Machine)**:
  - 精确管理 `idle` -> `thinking` -> `tool_calling` -> `answering` 状态。
  - 状态与 UI 自动联动（Vue 响应式优势）。
- **会话持久化 (Persistence)**:
  - 会话列表 CRUD。
  - 消息历史记录保存 (Prisma + PostgreSQL)。
- **Markdown 渲染器**:
  - 支持代码高亮、公式渲染。
  - 优化流式传输下的未闭合代码块处理。

### 2. 简历亮点功能 (Highlights)

这些功能不仅是核心，更是展示技术深度的关键点：

- **高性能虚拟滚动**: 处理万级消息列表，保证滑动丝滑。
- **并行工具调度器**: 支持 AI 同时调用多个工具（如搜索 + 生成），并实时展示各工具进度。
- **响应式流式解析器**: 自定义 SSE 解析逻辑，优化渲染频率，降低 CPU 占用。

### 3. 安全性与防盗刷 (Security & Rate Limiting)

作为生产级应用，安全性是重中之重，也是面试中的加分项：

- **API Key 保护**:
  - **服务端代理**: 所有 AI 请求必须通过 Nuxt Server API 转发，前端严禁直接持有或调用 API Key。
  - **环境变量加密**: 敏感密钥通过 `.env` 管理，并结合 Zod 进行启动时校验。
- **流量控制与防盗刷**:
  - **速率限制 (Rate Limiting)**: 基于用户 ID 或 IP 实现接口限流（如：10 次/分钟，500 次/天）。
  - **Token 配额管理**: 限制单次对话的最大上下文长度，防止恶意请求消耗大量 Token。
  - **强认证校验**: 核心聊天接口必须经过 JWT 校验，拒绝匿名访问。

### 4. 可选功能 (Extended - 后续增强)

- 语音输入 (Web Audio API)。
- 消息导出 (PDF/Markdown)。
- 会话分享链接。

## 二、 技术栈选型 (Tech Stack)

| 领域           | 技术方案                        | 选型理由                                             |
| :------------- | :------------------------------ | :--------------------------------------------------- |
| **前端框架**   | **Vue 3 (Composition API)**     | 响应式追踪更精确，适合复杂状态机。                   |
| **全栈框架**   | **Nuxt 4**                      | 约定优于配置，内置 Server API 和自动导入，极速开发。 |
| **状态管理**   | **Pinia**                       | 简洁、类型安全，Vue 官方推荐。                       |
| **数据库/ORM** | **PostgreSQL + Prisma**         | 强类型 ORM，开发体验极佳。                           |
| **样式/UI**    | **Tailwind CSS + Element Plus** | Tailwind 负责布局，Element Plus 提供基础组件。       |
| **Markdown**   | **markdown-it + Shiki**         | 高度可定制，代码高亮效果出色。                       |

## 四、 核心产出目标 (Core Deliverables)

1. **高性能聊天页**: 即使消息成千上万也不卡顿。
2. **可视化的推理过程**: 用户能清晰看到 AI 的思考过程和工具调用进度。
3. **健壮的全栈架构**: 类型安全的 API 和清晰的业务分层。
